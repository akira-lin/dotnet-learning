.NET 底层

GC 垃圾回收机制

代际垃圾回收

.NET 的 GC 把对象分三代：

代说明GC 频率
Gen 0短期对象最频繁
Gen 1中期对象中等
Gen 2长期对象很少

常见 GC 问题

  • 大对象直接进大对象堆（>85KB）
  • 过度装箱/拆箱
  • Finalize 拖慢 GC

性能观察工具

# 查看 GC 行为 dotnet-gcviz # 性能计数器 dotnet-counters monitor # CPU Profiling dotnet-trace collect dotnet-trace report

优化建议

  • 避免频繁分配大对象
  • 使用对象池（ArrayPool）
  • 及时释放 IDisposable
  • Span<T> 减少分配
.NET 底层

IOC 容器底层原理

手写简单 IOC 容器

public class MyContainer { private readonly Dictionary<Type, Type> _services = new(); public void Register<TInterface, TImplementation>() where TImplementation : TInterface { _services[typeof(TInterface)] = typeof(TImplementation); } public TInterface Resolve<TInterface>() { var interfaceType = typeof(TInterface); var implType = _services[interfaceType]; // 获取构造函数 var ctor = implType.GetConstructors()[0]; var parameters = ctor.GetParameters(); // 递归解析依赖 var args = parameters .Select(p => Resolve(p.ParameterType)) .ToArray(); return (TInterface)ctor.Invoke(args); } }

生命周期

方式实现场景
Singleton全局一个实例配置、缓存
Scoped每请求一个DbContext
Transient每次新实例轻量服务

循环依赖问题

⚠️ 构造函数循环依赖会导致栈溢出：
A 依赖 B，B 依赖 A
解决方案：改用属性注入或重构设计
.NET 底层

异步编程深层

ValueTask vs Task

// ValueTask：避免同步完成的包装开销 public async ValueTask<User> GetUserAsync(int id) { var cached = _cache.Get(id); if (cached != null) return cached; // 同步返回，避免包装 return await _db.Users.FindAsync(id); }

使用场景

类型适用场景
Task大部分情况
ValueTask高性能，热路径

IValueTaskSource（进阶）

用于高性能场景的自定义异步源：

// 一般项目用不到，但面试可能问 public class MyAsyncSource : IValueTaskSource<T> { // 实现 GetResult、GetStatus、OnCompleted }

常见误区

❌ 不要 async void（除事件处理）
❌ 不要在 using 里 await（行为复杂）
✅ 优先使用 using (await ...)
.NET 底层

内存管理进阶

Span<T> 和 Memory<T>

// Span：栈上分配，避免 GC Span<byte> span = stackalloc byte[1024]; // 处理数组片段 var slice = array.AsSpan(10, 100); slice.Clear(); // ReadOnlySpan 用于只读 ReadOnlySpan<char> text = "hello".AsSpan();

ArrayPool

var pool = ArrayPool<byte>.Shared; var buffer = pool.Rent(1024); try { // 使用 buffer } finally { pool.Return(buffer); // 归还，不触发 GC }

Unsafe 工具类

// 高性能场景，谨慎使用 var ptr = Unsafe.Alloc<byte>(size); Unsafe.Free(ptr);

stackalloc 安全问题

⚠️ stackalloc 在托管栈上分配
❌ 不要返回 Span 到栈帧外
❌ 避免大数组 stackalloc（栈溢出）

何时使用？

高性能场景（序列化/加密/解析），普通业务代码不需要。

分布式

CAP 定理

分布式系统最多同时满足两个：

三选二

C（一致性）：所有节点数据一致 A（可用性）：每次请求都有响应 P（分区容错）：网络分区时仍运行 CA（不可能）：单机数据库 CP（大多数选择）：Redis、MongoDB AP（大多数选择）：Eureka、Cassandra

实际取舍

场景选择例子
注册中心APEureka
分布式缓存CPRedis Cluster
分布式数据库CPMySQL Group Replication
消息队列APKafka
