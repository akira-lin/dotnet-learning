## 📌 阶段二注意事项

1. **Redis 不是万能的**
   - 先搞清楚什么时候用缓存，什么时候不用
   - 滥用缓存会导致数据不一致

2. **Docker 优先于 K8s**
   - 先把 Docker 用熟
   - K8s 是下一个阶段的事

3. **微服务不是银弹**
   - 小项目用微服务反而增加复杂度
   - 知道什么时候用，比会用更重要

4. **开始注重架构思维**
   - 不只是实现功能，还要想"怎么设计更合理"
   - 试着自己画架构图
