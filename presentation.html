Week 1-2：.NET 基础 + 第一个 WebAPI

必须掌握的

  • .NET 9 SDK 安装和项目创建
  • ASP.NET Core WebAPI 项目结构
  • HTTP 协议基础（GET/POST/PUT/DELETE）
  • JSON 数据格式
  • Postman 测试接口
  • 依赖注入（会用即可）

目标

能创建 WebAPI 项目
能写增删改查接口
能用 Postman 测试

完成作品：TodoList API

Week 3-4：数据库操作

EF Core + SQLite

必须掌握的

  • EF Core 增删改查
  • SQLite 数据库（轻量，无须安装）
  • DTO 数据传输对象
  • 异步编程（async/await）
  • 分页、排序、模糊查询

为什么用 SQLite？

  • 无需安装数据库服务
  • 整个数据库就是一个文件
  • 适合学习和小型项目
  • 部署简单，拷贝文件即可

Week 9-12：WebAPI 进阶 + Redis

必须掌握的

  • Redis 缓存（面试亮点）
  • RabbitMQ 消息队列
  • JWT 鉴权进阶
  • 接口限流
  • Nginx 反向代理

Redis 适用场景

  • 热点数据缓存
  • 会话存储
  • 消息队列
  • 分布式锁

Week 13-16：Docker + 微服务

Docker

  • 容器化部署
  • Docker Compose 多容器
  • 镜像构建
  • 基本 Linux 命令

微服务组件

  • 服务注册与发现（Consul）
  • API 网关（Ocelot）
  • 熔断降级（Polly）
  • 链路追踪（SkyWalking）

阶段三：架构师路线

核心优先

  • .NET 底层（GC/IOC/异步）
  • 设计模式 + 架构思维
  • 微服务完整技术栈
  • 分布式 + 高并发

按需学习

  • K8s（有项目需求时）
  • Elasticsearch（搜索需求）
  • Kafka（大数据量）
  • Azure（出海/外企）

架构师 vs 中级工程师

方面中级工程师架构师
问题解决解决具体问题定义问题边界
技术选型会用现有技术决定用什么技术
系统设计能看懂架构图能画架构图并讲清为什么
业务理解理解自己负责的模块理解整个系统业务流
技术深度知道怎么用知道底层原理
